# Contech DX UI/UX 개선 전략

## 1. 개요 (Executive Summary)
**Contech DX** 플랫폼을 기능적인 프로토타입에서 프리미엄 엔터프라이즈급 건설 관리 시스템으로 격상시키는 것을 목표로 합니다. "전문적인 미니멀리즘(Professional Minimalism)"(OpenAI/Linear 스타일)에 초점을 맞추어, 복잡한 데이터를 깔끔하게 표현하고 끊김 없는 유연한 인터랙션을 제공합니다.

## 2. 현황 분석 (Current State Analysis)
### 강점 (Strengths)
- **최신 기술 스택**: Next.js 16, React 19, Tailwind v4로 탄탄한 기반이 마련되어 있습니다.
- **깔끔한 베이스**: `globals.css`에 이미 세련된 컬러 팔레트(뉴트럴 + 관리자 오렌지 + 강조 블루)가 정의되어 있습니다.
- **내비게이션**: 프로젝트 상세 페이지의 사이드바 구조는 복잡한 데이터를 다루기에 좋은 UX 패턴입니다.

### 약점 (Weaknesses)
- **일반적인 랜딩 페이지**: 메인 페이지(`/`)가 SaaS 대시보드라기보다는 마케팅용 랜딩 페이지 같은 느낌을 줍니다.
- **빈 상태(Empty States)**: 프로젝트 상세의 "작업(Tasks)", "팀(Team)", "문서(Documents)" 탭이 비어 있습니다.
- **시각적 위계**: 리스트와 카드가 주요 정보와 보조 정보를 명확하게 구분하지 못하고 있습니다.
- **인터랙션 피드백**: 일부 영역에서 미세한 호버 상태, 전환 효과, 로딩 스켈레톤 등이 부족합니다.

## 3. 디자인 원칙 (Design Principles)
1.  **전문적인 미니멀리즘**: 과도한 배경색 대신 여백과 미세한 테두리를 사용합니다. 다크 모드에서는 완전한 검정(`#000000`)을 사용합니다.
2.  **데이터 밀도**: 건설 관리는 많은 데이터를 한눈에 봐야 합니다. 컴팩트한 테이블과 그리드를 사용하되 가독성을 유지합니다.
3.  **맥락적 내비게이션**: 사이드바는 맥락(전역 vs 프로젝트별)에 따라 적응해야 합니다.
4.  **유연한 모션**: `framer-motion`을 사용하여 레이아웃 변경(예: 사이드바 열기, 탭 전환) 시 앱이 "살아있는" 느낌을 주도록 합니다.

## 4. 실행 계획 (Action Plan)

### 1단계: 기반 및 디자인 시스템 (Foundation & Design System)
-   **타이포그래피 정제**: `Inter` 또는 유사한 가변 폰트를 사용하고 제목의 자간을 좁혀 세련미를 더합니다.
-   **컴포넌트 표준화**: `src/components/ui`를 점검하여 모든 컴포넌트(버튼, 입력창, 카드)가 `globals.css`의 토큰을 엄격히 따르도록 합니다.
-   **다크 모드 폴리싱**: 다크 모드에서 모든 테두리와 배경의 대비가 적절한지 확인합니다.

### 2단계: 대시보드 전환 (Dashboard Transformation - `/`)
-   **대시보드화**: 로그인한 사용자에게는 "히어로 섹션" 대신 **프로젝트 개요 대시보드**를 보여줍니다.
    -   **현황 행**: 전체 프로젝트, 진행 중인 현장, 총 예산, 이슈 건수.
    -   **최근 활동**: 최근 업데이트 피드 (예: "프로젝트 A 상태가 진행 중으로 변경됨").
    -   **빠른 작업**: "새 프로젝트", "문서 검색".
-   **게스트용 랜딩 유지**: 로그인하지 않은 사용자에게만 마케팅 히어로 섹션을 보여줍니다.

### 3단계: 프로젝트 상세 및 Gantt 통합 (Project Detail & Gantt Integration)
-   **"작업(Tasks)" 탭 활성화**: 이곳에 Gantt 차트 컴포넌트를 통합합니다. 라이브러리가 외부에 있다면 컨테이너와 인터페이스를 설정합니다.
-   **사이드바 개선**:
    -   "단계(Phases)" 또는 "WBS"를 위한 접이식 섹션을 추가합니다.
    -   프로젝트 이름 옆에 진행률 표시기(예: 원형 그래프)를 추가합니다.
-   **"대시보드" 탭**: S-Curve 및 자원 사용량을 보여주는 차트(`recharts` 등 활용)로 "프로젝트 개요" 탭을 구현합니다.

### 4단계: 마이크로 인터랙션 (Micro-Interactions)
-   **리스트/그리드 전환**: 프로젝트 목록과 그리드 뷰 간의 전환을 애니메이션화합니다.
-   **호버 효과**: 카드에 `group-hover` 효과를 추가하여 편집/삭제 같은 작업 버튼이 필요할 때만 나타나도록 하여 시각적 복잡도를 줄입니다.
-   **스켈레톤 로딩**: `ProjectList`와 `ProjectDetail`에 일치하는 스켈레톤 상태를 구현하여 레이아웃 이동(Layout Shift)을 방지합니다.

## 5. 기술적 제언 (Technical Recommendations)
-   **아이콘**: `lucide-react`를 계속 사용하되 일관된 스트로크 두께(1.5px 또는 2px)를 유지합니다.
-   **레이아웃**: 복잡한 대시보드의 반응형 구현을 위해 CSS Grid를 적극 활용합니다.
-   **상태 관리**: Gantt 상태가 복잡해질 경우, 전역 UI 상태(사이드바 토글, 뷰 모드 등) 관리를 위해 `zustand` 도입을 고려합니다.
